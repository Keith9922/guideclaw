# 引路虾 GuideClaw 三到四分钟演示讲稿

> 当前演示默认基座模型：`openrouter/free`  
> 说明：Web 工作台和 FastAPI 后端通过 OpenRouter 兼容接口接入模型；OpenClaw 负责角色执行和 skills 调用。实际部署时可通过 `OPENROUTER_MODEL` 替换为其他模型。

## 讲稿

### 0:00 - 0:20 开场

大家好，我的项目叫 **引路虾 GuideClaw**。

它是一个面向科研新人的研究导航系统。它想解决的，不是单纯帮你翻译一篇论文，而是帮助一个对陌生领域并不熟悉的用户，从一个模糊课题出发，逐步走到文献梳理、研究缺口发现、方案生成，以及最终可交付的结果汇总。

今天我演示的案例是：**机器学习辅助 OLED 材料筛选**。

### 0:20 - 0:55 先展示结果汇总页

我先不从内部流程开始，而是先展示最终交付物。

这是引路虾的 **结果汇总页**。这里会把当前课题最有价值的结果集中整理出来，包括：

- 这个方向到底在研究什么
- 当前领域发展到什么程度
- 最推荐先读哪些论文
- 哪个切入点最适合继续推进
- 下一步应该怎么做

对于科研新人来说，这一页很重要，因为他们最缺的往往不是“多一个聊天窗口”，而是一份可以直接拿来阅读、理解和继续推进的结果。

这里同时支持网页直接查看、PDF 在线预览和 PDF 下载导出，所以它既可以在系统里看，也可以直接拿去做开题、组会或者和导师沟通。

### 0:55 - 1:40 切到工作区，讲项目作用和操作方式

接下来我切到工作区。

这里是引路虾的核心操作界面。用户只需要从一个课题开始，比如输入研究主题、补充背景说明，或者上传 PDF，系统就会开始做首轮调查。

引路虾真正解决的是科研新人最容易卡住的那一段：

- 不知道从哪里开始查
- 看了很多文献但理不清
- 不知道什么值得研究
- 有想法但不知道怎么收成一版方案

所以它服务的是“**从不会做，到能开始做**”这个过程。

### 1:40 - 2:20 讲它的核心特点：PI 编排 + 多 Agent 协作

引路虾不是一个单独的聊天机器人，它的核心特点是：**由一个 PI，也就是课题负责人，负责向下编排和分发任务。**

PI 会先基于当前课题，收敛出：

- 当前研究焦点
- 为什么这个问题值得现在做
- 当前最关键的几个问题
- 接下来优先调度哪个 Agent

然后再把任务分派给下面几个角色：

- 文献助理：负责找文献、整理证据、生成文献卡
- 选题分析员：负责识别研究缺口
- 方案设计师：负责形成研究方案
- 组会秘书：负责沉淀纪要、待办和下一步

所以它不是多个 prompt 轮流说话，而是一套围绕科研课题运转的工作流。

当前演示版默认使用的基座模型是 **`openrouter/free`**，但这个模型并不是产品本体。真正重要的是：PI 如何编排、知识库如何沉淀、以及最后如何交付结果。

### 2:20 - 2:55 讲它和普通 OpenClaw 聊天 / Skills 的区别

这里我想强调一个差异。

如果只是一次性拿到一个答案，当然也可以直接在 OpenClaw 聊天里去调 skills。但引路虾不是这样设计的。

它和普通技能调用型项目最大的区别在于：

- 它是项目级的，而不是单轮对话
- 它有项目知识库
- 它有结构化成果卡
- 它支持继续追问和持续推进
- 它最后会收成一个结果汇总和 PDF 交付物

所以 OpenClaw 在这里更多承担的是底层执行能力，而引路虾真正提供的是一个面向科研新人的产品系统。

### 2:55 - 3:25 切到知识库，讲数据准确性和 PDF 能力

接下来我切到知识库页。

这里是项目级知识库。用户上传的 PDF、系统检索到的论文源、知识块、原始链接和 DOI，都会进入当前项目里。

这样做的目的，是尽量保证系统输出不是凭空生成，而是有来源、有引用、可回溯。比如搜索文献的结果，会附带原始链接；后续成果卡和结果汇总，也会尽量回到这些知识源。

另外，如果页面里有英文内容，系统支持局部翻译，这对科研新人会更友好。

### 3:25 - 3:45 切到角色页，讲技术实现

最后我快速看一下角色执行页。

从技术上看，这个项目采用的是：

- Web 工作台负责交互和最终交付
- FastAPI 后端负责项目管理、知识库和工作流编排
- OpenClaw 负责角色执行和 skills 调用
- SQLite 负责项目和成果的持久化存储

因此它不是一个前端壳子，也不是单纯拼 skill，而是一个完整的、多 Agent 协作的科研工作系统。

### 3:45 - 4:00 收尾

总结一下：

引路虾面向的是 **不懂某个领域的科研新人**。它解决的问题不是“回答一篇论文写了什么”，而是：

- 这个领域到底是什么
- 什么值得研究
- 我应该先看什么
- 我该怎么继续推进

最终，它把一个陌生课题，收成了一个可持续推进的科研项目，以及一份可直接查看、可导出、可继续追问的结果汇总。

这就是我的项目，谢谢大家。
